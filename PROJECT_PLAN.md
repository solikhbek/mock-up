# FastFood POS System — Project Plan

## 📋 Общая информация

| Параметр | Значение |
|----------|----------|
| Название | FastFood POS (рабочее название) |
| Тип проекта | MVP для реального бизнеса |
| Регион | Узбекистан |
| Количество филиалов | 2-5 |
| Каналы продаж | Зал + Самовывоз |

---

## 🎯 Цели проекта

1. Полная цифровизация админ-части фаст-фуда
2. Единая система управления несколькими филиалами
3. Прозрачный учёт финансов и склада
4. Ускорение обслуживания клиентов через KDS/CDS

---

## 🧩 Модули системы

### 1. POS (Касса)
- Приём заказов
- Модификаторы блюд (без лука, extra сыр и т.д.)
- Комбо-наборы
- Разделение счёта
- Скидки и промокоды
- Интеграция с платёжными системами:
  - **Uzcard**
  - **Humo**
  - **Visa/Mastercard**
  - **Click**
  - **Payme**
  - **Uzum Bank**

### 2. KDS (Kitchen Display System) — Экран кухни
- Отображение заказов в реальном времени
- Статусы: Новый → Готовится → Готов
- Таймер приготовления
- Цветовая индикация (зелёный/жёлтый/красный по времени)
- Группировка по станциям (гриль, фритюр, напитки)

### 3. CDS (Customer Display System) — Экран клиента
- Номер заказа
- Статус: "Готовится" / "Готов"
- Звуковое оповещение
- Очередь заказов

### 4. Склад (Inventory)
- Номенклатура товаров/ингредиентов
- Приход товаров от поставщиков
- Автоматическое списание при продаже (по техкартам)
- Инвентаризация
- Минимальные остатки + уведомления
- Перемещение между филиалами

### 5. Финансы (Приход/Уход)
- Учёт выручки по кассам
- Инкассация
- Расходы (закупки, зарплаты, аренда, прочее)
- P&L отчёт по филиалам
- Сверка с платёжными системами

### 6. Отчёты и Дашборды
- **Дашборд руководителя:**
  - Выручка сегодня/неделя/месяц
  - Сравнение филиалов
  - Топ продаж
  - Средний чек
- **Операционные отчёты:**
  - Продажи по часам (для планирования смен)
  - ABC-анализ меню
  - Отчёт по отменам/возвратам
  - Время обслуживания
- **Складские отчёты:**
  - Остатки
  - Движение товаров
  - Себестоимость

### 7. Управление филиалами
- Список филиалов с адресами
- Настройки каждого филиала
- Централизованное меню (синхронизация)
- Локальные цены (если нужно)

### 8. Меню
- Категории блюд
- Карточки блюд (фото, описание, цена)
- Техкарты (рецептуры для списания)
- Модификаторы
- Комбо-конструктор
- Стоп-лист

## 👥 Роли и права доступа

| Роль | Права |
|------|-------|
| **Супер-админ** | Полный доступ ко всей системе и всем филиалам |
| **Владелец/Директор** | Все отчёты, финансы, настройки меню, просмотр всех филиалов |
| **Управляющий филиала** | Отчёты своего филиала, управление сменами, склад филиала |
| **Кассир** | Только касса, просмотр своих продаж |
| **Повар** | Только KDS (экран кухни) |
| **Кладовщик** | Приход товаров, инвентаризация, остатки |
| **Бухгалтер** | Финансовые отчёты, выгрузки для 1С |

## 🔄 Бизнес-процесс заказа

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         CUSTOMER JOURNEY                                     │
└─────────────────────────────────────────────────────────────────────────────┘

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Клиент  │───▶│  Кассир  │───▶│  Кухня   │───▶│  Экран   │───▶│  Выдача  │
│ приходит │    │ принимает│    │ готовит  │    │ клиента  │    │  заказа  │
│          │    │  заказ   │    │          │    │ (готов!) │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                     │
                     ▼
              ┌──────────────┐
              │   Оплата:    │
              │ • Uzcard     │
              │ • Click      │
              │ • Payme      │
              │ • Uzum Bank  │
              │ • Visa/MC    │
              │ • Наличные   │
              └──────────────┘
```

### Детальный flow:

1. **Клиент подходит к кассе**
2. **Кассир создаёт заказ:**
   - Выбирает блюда из меню
   - Добавляет модификаторы
   - Применяет скидку/промокод (если есть)
3. **Оплата:**
   - Выбор способа оплаты
   - Интеграция с терминалом/QR
   - Печать чека (фискальный)
4. **Заказ уходит на кухню (KDS):**
   - Отображается на экране поваров
   - Статус: "Новый"
5. **Повар начинает готовить:**
   - Нажимает "Начать" → статус "Готовится"
   - На CDS клиент видит свой номер
6. **Заказ готов:**
   - Повар нажимает "Готово"
   - На CDS номер переходит в "Готов"
   - Звуковой сигнал
7. **Выдача:**
   - Клиент подходит, забирает заказ
   - Заказ закрывается

## 🛠️ Технологический стек (рекомендация)

### Frontend
- **Web Admin Panel:** React / Next.js
- **POS (Касса):** Electron + React (для offline режима)
- **KDS/CDS:** React (веб на планшетах/ТВ)

### Backend
- **API:** Node.js (NestJS) или Python (FastAPI)
- **База данных:** PostgreSQL
- **Кэш:** Redis
- **Очереди:** Redis / RabbitMQ

### Интеграции
- **Платежи:** Click API, Payme API, Uzum API
- **Фискализация:** Интеграция с ОФД Узбекистана
- **1С:** REST API для выгрузки

### Инфраструктура
- **Хостинг:** Cloud (AWS/DigitalOcean) или локальный сервер
- **CI/CD:** GitHub Actions
